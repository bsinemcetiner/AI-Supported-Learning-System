Text Processing
●
Printer Output: On UNIX-like systems, output destined for a printer is sent 
as plain text or, if the page contains graphics, is converted into a text format 
page description language, known as PostScript, which is then sent 
to a program that generates the graphic dots to be printed. 
●
Program source code: Many of the command line programs found on 
UNIX-like systems were created to support system administration and 
software development, and text processing programs are no exception. 
●
Many of them are designed to solve software development problems. 
●
The source code is always in text format.

Text Processing: uniq
●
Compared to sort, the uniq program is a lightweight.
●
It performs seemingly trivial task.
●
When given a sorted file, including the standard output, it removes any 
duplicate lines and sends the results to the standard output. 
●
It is often used in conjunction with sort to clean the output of duplicates. 


Text Processing: uniq
●
The uniq program has interesting options.
●
For example, -c option outputs a list of duplicate lines preceded by the 
number of times the line occurs.
●
The -d option only outputs repeated lines, rather than unique lines.

Text Processing: diff
●
The diff program is one of the most common tools used by UNIX developers.
●
Developers usually receive patch files for bug fixes.
●
The diff and patch programs are used together.
●
Let’s see how they work. 


Text Processing: diff
●
diff has three types of output. 
●
The default one has the following change commands. 
○
p1ap2, p1cp2, p1dp2
●
Notice that, even though they are written in one piece, they contain three 
fields. 
●
The first field is the position in the first file (p1), the second is the command, 
and the third is the position in the second file (p2). 


Text Processing: diff
●
p1ap2 : The a is for the append command. Append the lines at the position 
p2 in the second file to the position p1 in the first file.
●
p1cp2 : The c is for change, or replace. Change the lines at the position p1 
with the lines at the position p2 in the second file.
●
p1dp2 : The d is for delete. Delete the lines in the first file at position p1, 
which would have appeared at range p2 in the second file. 
●
It is also possible to use the context format with the option -c. 


Text Processing: diff
●
The context format begins with the names 
of files and their timestamps. 
●
The first file is marked with asterisks, the 
second file with dashes.
●
Then there are indicators for each line. 


Text Processing: diff
●
If there are no indicators, there is no 
difference. 
●
A minus sign (-) indicates that a line is 
deleted. 
●
A plus sign (+) indicates that a line is 
added.
●
An exclamation mark (!) indicates that a 
line is changed. Two versions of the line 
will be displayed, each in its respective 
section of the change group. 


Text Processing: diff
●
The unified format, -u, is similar to context 
format but is more concise.
●
Only the minus and plus signs are used to 
indicate the changes. 
●
If there is no change, no indicator is used.
