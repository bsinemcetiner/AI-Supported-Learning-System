awk
● Identifying awk as a programming language scares some people away from 
it. 
● Consider awk a different approach to problem solving. 
● The key to awk’s operations is that the data it works on has some kind of 
structure. 
● A text file can be loosely or tightly structured. 
○ A chapter containing major and minor sections has some structure. 
○ A table consisting of tab-separated items in columns is very structured. 
○ We can use an awk script to reorder columns of data, for example. 
awk
● Some of the things awk allows us to do:
○ View a text file as a textual database made up of records and fields. 
○ Use variables to manipulate the database. 
○ Use arithmetic and string operators. 
○ Use common programming constructs such as loops and conditionals. 
○ Generate formatted reports. 
○ Define functions.
○ Execute UNIX commands from a script. 
○ Process the result of UNIX commands. 
○ Process command-line arguments more gracefully. 
○ Work more easily with multiple input streams. 
awk
● Alfred V. Aho, Brian W. Kernighan and Peter J. Weinberger define awk as 
follows: 
○ Awk is a programming language whose basic operation is to search a set of files for patterns, 
and to perform specified actions upon lines or fields of lines which contain instances of those 
patterns.
○ The basic operation of awk is to scan a set of input lines in order, searching for lines which 
match any of a set of patterns which the user has specified.
○ For each pattern, an action can be specified; this action will be performed on each line that 
matches the pattern.

awk: usage
● The command 
awk program [files] 
Executes the awk commands in the string “program” on the set of named 
files, or on the standard input, if there are no files. 
● It is also possible to place the statements in a program file

awk: usage

awk ‘pattern’ [fileName]
awk ‘{ action }’ [fileName]
awk ‘pattern { action }’ [fileName]

An overview on how awk works
● Let's dissect the following command: awk '{ print $1, $3 }' myFile.in
● awk takes a line of input from the file myFile.in and puts the line into an 
internal variable called $0 (zero). Each line is also called a record.
● Next, the line is broken into fields or words separated by a whitespace 
character. Each field is stored in a numbered variable, starting with $1.
● awk knows about the whitespace characters according to FS, field separator, 
which may be changed.
myFile.in:
Kaya A209 8289 CE
İlker A421 8258 CE

awk’s Programming Model
● It’s important to understand the basic model that awk offers the programmer. 
● An awk program consists of what we will call a main input loop. 
● We don’t write this loop, it exists as the framework within which the code we 
write will be executed. 
● The main input loop in awk is a routine that reads one line of input from a file 
and makes it available for processing. 
● The actions we write to do the processing assume that there is a line of input 
available. 
awk’s Programming Model
● In another programming language, we would have to create the main input 
loop as part of our program. 
● This program would have to open the input file, and read one line at a time. 
● This is not a lot of work, but it illustrates a basic awk shortcut that makes it 
easier for us to write our programs. 
● The main input loop is executed as many times as there are lines of input. 
● It terminates when there is no more input to be read. 
awk’s Programming Model
● awk allows us to write two special routines that can be executed before any 
input is read and after all input is read. 
● These are the procedures associated with the BEGIN and END rules. 
● In other words, we can do some preprocessing before the main input loop is 
ever executed, and some post processing after the main input loop is 
terminated. 
● The BEGIN and END procedures are optional.

awk’s Processing Model
● So, we can think of an awk script as having three major parts, what happens 
before, what happens during, and what happens after processing the input.
● The “during” process is where most of the work gets done. 
● Inside the main input loop, our instructions are written as a series of pattern 
and action procedures. 
● A pattern is a rule for testing the input line to determine whether or not the 
action should be applied to it. 
● The actions can be quite complex, consisting of statements, functions and 
expressions.

awk: Records and Fields

Either the colon, the tab, or the space can separate the fields.

Programming with awk
● There are two types of constants in awk; string or numeric.
● A string must be quoted in an expression. Strings can make use of the 
escape sequences, such as \n, \r, \t.
● To define a variable, we only have to name it, and assign it a value. 
● The name can only contain letters, digits and underscores, and may not start 
with a digit. 
● The variable names are case sensitive. 
● Variables are not declared, we don’t have to tell what type of value will be 
stored in a variable. Each has a string and a numeric value, and awk uses the 
appropriate value based on the context of the expression. 
Programming with awk
● Variables do not have to be initialized, awk automatically initializes them to 
the empty string, which acts like 0 if used as a number. 
● The following expressions assign values to x and z. 
x = 1
z = "Hello"
● A space is the string concatenation operator. The expression 
z = "Hello" "World"
returns "HelloWorld"

System variables
● There are a number of system or built-in variables defined by awk. 
● awk has two types of system variables. 
● The first one defines values whose default can be changed, such as the 
default field and record separators.
● The second type defines values that can be used in reports or processing, 
such as the number of fields found in the current record, the count of the 
current record. These are automatically updated by awk. 
System variables
● You remember FS, is the field separator. You also remember its output 
equivalent, OFS. 
● awk defines the variable NF to be the number of fields for the current input 
record. You can use $NF, with $, to reference the last field of each record. 
● awk sets the variable NR to the number of the current input record. After a 
record has been processed, NR is incremented.
● awk also defines RS, the record separator, as a new line. Its output 
equivalent is ORS. 
Relational and Boolean operators
● Relations and Boolean operators allow us to make 
comparisons between two expressions.
● A list is given in the table on the right.
● A relational expression can be used in place of a 
pattern to control a particular action. For example, if 
we want to limit the records selected for processing 
to those that have six fields, we could use the 
following expression 
NF == 6 { print $1, $6 }

Relational and Boolean operators
● We can use the match operator, ~ for fields.
$5 ~ /MA/ { print $1, ", " $6 }
● It also works with variables; 
$5 ~  state { print $1, ", " $6 }
Relational and Boolean operators
● Given two or more expressions, || can be used for logical OR, && can be 
used for logical AND. 
NF == 6 && NR > 1
● && has higher precedence than ||.

Passing parameters into a script
● One of the more confusing subtleties of awk programming is passing 
parameters into a script.
awk -v myVar=$HOME { print myVar } anyfile
Conditionals
● The conditional statements in awk are borrowed from C, just as Java. 
● They are used to control the flow of the program in making decisions. 
awk '{ if ($5 > 50) print $1 }' filename
awk '{ if ($4 > 50) print $1; else print $2 }' filename
● Of course, it is better to write it as a separate awk script, if the script contains 
many if, else if, and else statements. 
● awk also borrows the following operator from C, just as Java. 
expression ? action1 : action2 
Looping
● awk supports while, do while and for loops. 
● We can use break and continue to break the loop, or skip remaining 
statements.
● while loop works as follows:
i = 1
while ( i <= 4 ) {
    print $i
    ++i
}
